Ahead-Of-Time (AOT) Zero-Reflection Type Validation

Webergency Typechecker

Webergency Typechecker is a high-performance, zero-runtime-dependency TypeScript compiler plugin (transformer). It intercepts standard TypeScript type definitions and converts them directly into optimized, hashed, and hoisted JavaScript validation functions at build time.

By generating static pipelines, it completely bypasses runtime reflection, parsing schemas, or dynamic validator evaluation, achieving execution speeds up to 10x faster than standard runtime checkers.

⚡

Compiled Pipelines

Translates your types directly into vanilla JavaScript checks, keeping the runtime package size at a absolute zero dependency footprint.

🏷️

Tag-Based Modifiers

Impose constraints, formatting rules, transformations, and default values directly onto TypeScript primitives using intersection types.

Installation

Since this library is a TypeScript compiler plugin (transformer), you must compile your project using a compiler patcher like ts-patch to hook into TS compilation.

Terminal 
npm install @webergency-utils/typechecker
npm install -D ts-patch

# Run patcher to inject the compiler hook
npx ts-patch install

Configuration

Add the typechecker plugin to the plugins array inside your project's tsconfig.json:

tsconfig.json 
{
  "compilerOptions": {
    "target": "ES2022",
    "module": "NodeNext",
    "moduleResolution": "NodeNext",
    "plugins": [
      { "transform": "@webergency-utils/typechecker" }
    ]
  }
}

Validation APIs

Import one of the validation helpers and pass any TypeScript type directly as a generic argument. The transformer will compile your call into a dedicated runtime validator.

validation.ts 
import { is, assert, validate } from '@webergency-utils/typechecker';

interface User {
  id: string;
  age: number;
}

const data: unknown = { id: "user_1", age: 24 };

// 1. is<T>() - returns a boolean Type Guard
if (is<User>(data)) {
  console.log(data.id); // TypeScript narrows type to User here
}

// 2. assert<T>() - returns value or throws a formatted Error
const validatedUser = assert<User>(data);

// 3. validate<T>() - returns structured detailed errors list
const result = validate<User>(data);
if (!result.success) {
  console.error(result.errors); // [{ path: "age", expected: "number", value: ... }]
}

WithModifiers Utility

If you have deeply nested objects or external types, you can add validation tags to specific properties using dot-notation path references. This prevents you from having to retype the entire structure.

deep-decorators.ts 
import { WithModifiers, constraint, format } from '@webergency-utils/typechecker';

// External type we cannot edit directly
interface ExternalUser {
  id: string;
  profile?: {
    details: {
      email: string;
      password: string;
    }
  }
}

// Decorate specific paths using dot notation
type ValidatedUser = WithModifiers<ExternalUser, {
  'profile.details.email': format.Email;
  'profile.details.password': constraint.MinLength<8>;
}>;
Automatic Type Merging

WithModifiers automatically intersects validation metadata with the original type at that path. For example, profile.details.email will be typed as string & Format<'email'> under the hood. It also preserves optional properties (?) and handles recursive types correctly.

Cross-Field Dependencies (Requires)

You can specify that a property requires the presence of other properties in the same object using absolute paths or relative dot-notation paths (e.g. .sibling, ..grandparent.cousin):

cross-dependencies.ts 
import { WithModifiers, constraint } from '@webergency-utils/typechecker';

interface DBConfig {
  ssl?: boolean;
  cert?: string;
  auth?: {
    username?: string;
    password?: string;
  }
}

type SecureConfig = WithModifiers<DBConfig, {
  // 1. Absolute check: if "ssl" is defined, "cert" must exist at root
  'ssl': constraint.Requires<'cert'>;

  // 2. Relative check: if "auth.password" is defined, its sibling "username" must exist
  'auth.password': constraint.Requires<'.username'>;
}>;

Namespace: constraint

Impose values constraints directly on your TypeScript types:

Tag / Modifier Applies To Description
constraint.MinLength<N> string Enforces a minimum string length.
constraint.MaxLength<N> string Enforces a maximum string length.
constraint.Length<Min, Max> string Combination helper to enforce string length range.
constraint.Pattern<S> string Validates string against a regex pattern.
constraint.Minimum<N> number | bigint Minimum value constraint (inclusive).
constraint.Maximum<N> number | bigint Maximum value constraint (inclusive).
constraint.Range<Min, Max> number | bigint Enforces that value lies within a numeric range.
constraint.MultipleOf<N> number | bigint Enforces value to be a multiple of N.
constraint.MinItems<N> Array Minimum number of elements in the array.
constraint.MaxItems<N> Array Maximum number of elements in the array.
constraint.UniqueItems Array Ensures all array elements are deeply unique.
constraint.Custom<typeof fn> Any Executes a custom validation function (receives (val, { parent, root, path }) => boolean).
constraint.Requires<Path | [Paths]> Any Enforces cross-field dependencies (supports absolute and relative sibling .sibling, grandparent ..cousin paths).

Namespace: format

Impose common format constraints on string primitives:

Format Tag Description / Validator
format.Email Standard RFC 5322 Email Validator.
format.UUID UUID (v1-v5) format.
format.URL Full URL validation (HTTP/HTTPS/FTP).
format.IPv4 / format.IPv6 Standard IP address validator.
format.DateTime ISO-8601 Date Time string.
format.Date Standard ISO Date string (YYYY-MM-DD).
format.ObjectId 24-character hexadecimal MongoDB ObjectId.
format.Duration ISO-8601 duration format (e.g., PT1H).

Namespace: transform

Modify input data before validating it:

Transform Tag Target Type Description
transform.Trim string Removes whitespace from both ends.
transform.LowerCase string Converts string to lowercase.
transform.UpperCase string Converts string to uppercase.
transform.Capitalize string Capitalizes the first character of the string.
transform.ToNumber Any Coerces string, boolean, etc. to number.
transform.ToBoolean Any Coerces values into boolean.
transform.ToDate Any Coerces string timestamp into a Date object.

Namespace: tag

Define default values inside your TypeScript types:

Tag Description
tag.Default<Value> Injects Value at validation time if the property is undefined.
defaults.ts 
import { tag, assert } from '@webergency-utils/typechecker';

interface Config {
  retries: number & tag.Default<3>;
  verbose: boolean & tag.Default<false>;
}

// Evaluates to: { retries: 3, verbose: false }
const finalConfig = assert<Config>({});
